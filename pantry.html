interview insights

skill

Many users can cook as long as instructions are simple and clear. Users would like step by step instructions with images or video. Research also showed that users would like to view and filter by recipe difficulty.

diet

A large majority of users noted dietary concerns or restrictions. This suggests that the app should allow users to avoid recipes containing allergens or intolerances while easily obtaining recipes that meet their dietary requirements.

quantity

The app should provide adjustable serving sizes to accomodate users and their widely varying family sizes.

cost

All users interviewed expressed interest in recipes that included prep and cook time as well as average cost per serving.

usability testing

home screen

User testing indicated that the Logout button should occupy less screen space. The Ingredients and Recipes buttons also occupy too much screen space, and could be minimized to make room for "Recipes of the Day" or other features.

ingredients search

Due to the lack of clear labels, users did not recognize the three icons at the top as quick filters.

ingredients search result

Many users tapped the Add Selected button before selecting the queried ingredient. This indicated that a prompt would aid in providing an easier user experience, as well as auto selecting a single ingredient if no others are returned.

select ingredients

Give users better visual clarification when they select an ingredient. Many users selected ingredients, but didn't recognize any change on the screen.

add ingredients

If users have not selected an ingredient, prompt them to do so if they tap the Add Selected button.

pantry

Most users preferred to query recipes by searching for ingredients and did not want or need the extra step of maintaining an inventory.

select pantry ingredients

User testing indicated that this screen was superflous, as they preferred not to manage an inventory. However, if this screen was kept, and if users had not selected an ingredient, I would have prompted them to do so if they tapped the Find Recipes button.

find recipes

Users would like more information at a glance, in addition to the name, picture, and cooking duration. Many users found it frustrating that when they tapped the Refine Ingredients button, they were taken back to the ingredients page without any prior searched ingredients shown or selected. The word refine did not accurately indicate what process would occur for the user upon tapping the Refine Ingredients button.

view recipe

The Favorite Icon, while originally designed to create space between the buttons that moved users foward from the buttons that took users back, nevertheless created visual discord which many users took note of. Users also indicated that the Favorite Icon should be filled if a recipe had already been added to their favorites.

my recipes

Users desired a way to organize, filter, and delete their favorited recipes.
